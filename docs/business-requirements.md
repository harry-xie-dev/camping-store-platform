# Business Requirements

## Business Goals

The simulated camping tableware store needs a simple data platform that can:

- Organize product, customer, and order data in one structured system.
- Support consistent tracking of order details and purchased items.
- Create a foundation for future customer and marketing analytics.
- Provide clear documentation for how business concepts map to data models.
- Demonstrate practical data platform design in a portfolio-friendly format.

## MVP Requirements

The minimum viable product should support the following capabilities when implementation begins:

- Maintain a product catalog with item names, categories, prices, and active status.
- Store simulated customer records with basic contact and profile information.
- Record customer orders with order dates, statuses, and totals.
- Track individual items within each order.
- Define marketing campaign records for future campaign attribution and analysis.
- Provide a backend API for core data access.
- Provide a frontend interface for viewing and managing store data.
- Support basic analytics queries for products, orders, customers, and campaigns.

## Out of Scope for Initial Documentation Phase

The current phase does not include:

- Backend application code.
- Frontend application code.
- Executable database migrations or a live database implementation.
- Real customer data.
- Real sales metrics or business results.
- Production deployment.

## Success Criteria

The initial documentation phase is complete when the repository clearly explains:

- The simulated business scenario.
- The target users.
- The main business goals.
- The MVP requirements.
- The proposed initial database tables.
- The intended purpose of each major project folder.
